# 搞定800词 · 800词资料格式建议

## 1. 推荐格式

优先推荐 CSV。原因：便于用户人工整理、便于检查缺字段、后端导入成本低，也方便后续用表格工具维护。

如果后续需要程序化维护，也可以提供 JSON。两种格式字段保持一致。

## 2. CSV 字段规范

文件名建议：`gaoding_800_words.csv`

| 字段名 | 必填 | 类型 | 说明 | 示例 |
|---|---|---|---|---|
| sort_no | 是 | number | 全局序号，1-800 | 1 |
| term | 是 | string | 成语词条 | 南辕北辙 |
| definition | 是 | string | 释义 | 比喻行动和目的相反。 |
| category | 建议 | string | 分类，可按主题或资料章节 | 逻辑关系 |
| example | 否 | string | 例句 | 做事要先明确目标，否则容易南辕北辙。 |
| source_page | 否 | number | 原 PDF 页码，便于追溯 | 3 |
| difficulty | 否 | string | 难度，easy/medium/hard | medium |
| tags | 否 | string | 多标签用英文逗号分隔 | 易混淆,高频 |

CSV 示例：

```csv
sort_no,term,definition,category,example,source_page,difficulty,tags

```

## 4. 整理规则

- `sort_no` 必须唯一，建议从 1 连续到 800。
- `term` 必须唯一，前后不要有空格。
- `definition` 必须填写，避免只写近义词。
- `category` 建议先使用资料原有章节；如果资料无分类，可统一填 `未分类`。
- `example` 可以为空，但建议保留，打卡页需要展示例句。
- 标点统一使用中文标点，英文逗号只用于 CSV 的 tags 分隔。
- 不要把多个成语合并在同一行。
- 不要在释义里写换行，避免 CSV 导入复杂化。
