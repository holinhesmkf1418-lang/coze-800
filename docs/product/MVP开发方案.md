# 搞定800词 · MVP 开发方案

## 1. 当前状态

截至 2026-06-06，第一轮 PM、前端、后端交付已进入可联调状态。

- 前端：原生微信小程序静态原型已完成，包含首页、打卡页、错题本、随心测、个人中心，TabBar 图标已补齐。
- 后端：Node.js + TypeScript + Express + Prisma 骨架已完成，包含登录、打卡、错题、随心测、导入接口，Prisma migration 已补齐。
- 数据：OCR 转换版词库已提交，当前结构化 755 条，可用于导入联调和数据校验，不作为最终生产词库。
- 文档：PRD、用户故事、测试用例、API 文档、部署方案已入库。

## 2. 开发目标

MVP 目标不是一次性做全功能，而是优先跑通“学习闭环”：

```text
微信登录
  -> 导入词库
  -> 每日打卡
  -> 随心测
  -> 自动沉淀错题
  -> 错题统计复盘
```

验收标准：

- 用户可以登录并获得后端 token。
- 后端可以导入 OCR CSV/JSON 词库，并能查询词汇分类。
- 打卡页使用真实接口展示今日词汇并提交打卡。
- 随心测由后端生成题目和选项，前端提交 selectedOption，后端判分。
- 答错词条进入错题本，错题统计能反映分类薄弱项。

## 3. 阶段拆分

### 阶段 0：联调准备

负责人：后端主导，前端配合，PM 验收。

交付内容：

- 后端提供本地启动说明、`.env.example`、数据库初始化和 migration 命令。
- 后端确认 `server/docs/api.md` 和实际代码一致。
- 前端新增统一 API 请求层，准备替换 mock。
- PM 确认接口字段与测试用例一致。

建议命令：

```bash
cd server
npm install
npx prisma generate
npx prisma migrate dev
npm run dev
```

验收口径：

- `GET /api/health` 可访问。
- `npm run build` 通过。
- 微信开发者工具可打开 `frontend/` 并编译。

### 阶段 1：登录与基础数据

负责人：后端 + 前端。

后端：

- 完成微信登录真实配置；本地开发可保留测试 openid 兜底方案。
- 完成词库导入接口联调。
- 完成分类接口 `/api/vocabs/categories`。

前端：

- 封装 `utils/request.js`。
- 封装登录态存取：token、用户信息、过期处理。
- 首页接入基础统计接口或先接 streak/categories。

验收口径：

- 首次进入触发登录。
- 登录成功后所有业务请求自动带 `Authorization: Bearer <token>`。
- OCR CSV 可以导入，导入结果返回成功数、失败数和失败行。

### 阶段 2：每日打卡闭环

负责人：前端主导，后端配合。

接口：

- `GET /api/check-in/today`
- `POST /api/check-in/submit`
- `GET /api/check-in/streak`

前端页面：

- 打卡页展示后端返回的词条、释义、分类、序号、例句。
- 提交打卡后页面状态切到已完成。
- 重复打卡展示后端返回提示，不新增记录。

验收口径：

- 正常打卡、重复打卡、跨天打卡三类测试用例通过。
- 首页能展示今日是否打卡和连续打卡天数。

### 阶段 3：随心测闭环

负责人：后端主导协议，前端主导交互。

接口：

- `POST /api/tests/start`
- `POST /api/tests/submit`
- `GET /api/tests/history`

协议口径：

- `start` 返回题目与 4 个选项：`options[{ label, text }]`。
- `start` 不返回正确答案。
- 前端提交：`answers[{ sortNo, selectedOption }]`，未答题传 `null`。
- 后端根据 DB 中的 `correctOption` 判分。
- 成绩页返回 `correctOption`、`isCorrect`、错题引用。

前端页面：

- 测试设置页：题量、倒计时。
- 答题页：顶部倒计时、进度条、词条、4 个释义选项。
- 成绩页：分数、正确/错误数、用时、错题明细。

验收口径：

- 随机抽题不重复。
- 前端看不到正确答案。
- 倒计时结束自动交卷。
- 成绩计算准确。
- 错题自动进入错题本。

### 阶段 4：错题分析闭环

负责人：前端主导页面，后端主导统计。

接口：

- `GET /api/wrong-answers`
- `GET /api/wrong-answers/stats`
- `POST /api/wrong-answers`
- `DELETE /api/wrong-answers/:vocabId`

前端页面：

- 错题总数、正确率、薄弱分类。
- 分类筛选。
- 错题列表展示词条、释义、错误次数、最近错误时间。
- 支持移出错题本。

验收口径：

- 随心测答错后错题出现。
- 同一词条重复答错只累加次数。
- 分类统计准确。

### 阶段 5：部署与第一轮验收

负责人：后端主导部署，前端提供小程序体验版，PM 组织验收。

交付内容：

- 后端 Docker 部署到测试环境。
- 前端配置测试环境 API 地址。
- 用户导入词库并跑通完整闭环。
- PM 按测试用例集执行验收。

验收口径：

- 测试环境可访问。
- 小程序体验版可完成登录、打卡、随心测、错题查看。
- 阻塞问题清零后进入下一阶段。

## 5. 风险与处理

| 风险 | 影响 | 处理方式 |
|---|---|---|
| OCR 词库只有 755 条 | 随心测 100 题可用，但不是最终数据 | 先用于联调，正式上线前人工校对补齐 |
| 微信登录依赖真实 AppID/Secret | 本地联调受阻 | 后端提供测试登录模式或 mock code |
| 随心测协议变更频繁 | 前端返工 | 以 `server/docs/api.md` 为唯一准绳，变更需同步 README |
| 数据库 schema 变更 | 部署失败 | 每次 Prisma schema 变更必须提交 migration |
| 前端一次性全量接线 | 排障困难 | 按登录、打卡、随心测、错题分阶段接入 |
